Empty Access Controls — The Primary Vulnerability
ACLs With No Role, No Condition, No Script
Critical
6,920
This is the exact vulnerability AppOmni documented. When an ACL has no Required Role, no Condition, and no Script, it defaults to allowing access to any user — including the guest (unauthenticated) user. On this instance, 6,920 ACLs match this pattern.
Not all of these are exploitable — many protect low-sensitivity system tables, and the effective access depends on table-level vs field-level ACL inheritance. But the pattern is the same one that caused data leaks on 70% of tested instances.

Remediation Priority
1
Review Guest User Account
The guest user is active. If unauthenticated access isn't required, deactivate the guest account. If it is required, ensure every table accessible via public portals has explicit ACLs restricting guest to only what's intended.
2
Audit Empty ACLs on Sensitive Tables
Of the 6,920 empty ACLs, prioritize those protecting user data (sys_user), credentials (oauth_entity), business data (incident, change_request), and configuration data (sys_properties). Add appropriate Required Roles to each.
3
Add ACLs to Custom Tables
The 45 custom tables with zero ACLs need at least a base read/write/delete ACL with an appropriate role. Focus on the 2 that have records — u_verb_test_576385 and u_verb_test_236571.
4
Restrict Public Portal Pages
Review the 19 public pages. Pages like "Request a Laptop" and "Employee Self Service" likely shouldn't be accessible without authentication. Remove the public flag or add authentication redirects.
5
Add ACLs to Client-Callable Script Includes
455 client-callable Script Includes lack ACLs. Prioritize those with "public" access and those handling sensitive operations (OAuthAccessToken, ConnectionAndCredentialHelper, password-related scripts).
6
Set Missing Security Properties
Explicitly set glide.script.block.client.globals, glide.cms.catalog.public_access (to false), and glide.ui.security.allow_coexistence (to false). Don't rely on platform defaults — they can change between versions.
7
Resolve Instance Scan Findings
Address the 554 deactivated security controls and 330 empty ACL findings flagged by ServiceNow's own scanner. These validate and overlap with our findings — resolving them addresses both assessments.
8
Configure Portal Login Pages
12 portals lack login page configuration. Add login redirects to ensure unauthenticated sessions are properly handled across all portal entry points.
