Performance Killers — Fix These First

These aren't dead — they're alive and actively wasting resources on every page load and every record save. They're the most impactful findings in this report.

Global Business Rules
79
Why this hurts: Global rules load into memory on every single page, every single transaction, for every single user. They exist as a legacy pattern from early ServiceNow — the platform recommendation is to migrate all of them to Script Includes, which only load when explicitly called.
Of the 79 global rules, most are OOB platform utilities (Role Delegation Functions, Get Number, Date Globals, Report Helpers). These can't be removed but represent a known platform overhead. Any customer-created global rules should be migrated to Script Includes immediately.

Before Rules Without Conditions on Busy Tables
71
Why this hurts: A Before rule with no filter condition evaluates on every insert, update, and delete on that table — even when the rule's logic doesn't apply. On high-volume tables like Incident (18 unconditioned rules) and User (17 unconditioned rules), these fire thousands of times daily. Each one blocks the user while it runs.

Note: Not all unconditioned Before rules are wrong — some intentionally run on every save (like "Get Number" or "Set Default Values"). The action item is to review each one and add filter conditions where the logic only applies to specific scenarios. Even adding current.isNewRecord() where appropriate cuts unnecessary evaluations dramatically.
